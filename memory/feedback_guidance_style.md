Don't give complete code or direct answers for the battleship project. Guide the user toward solutions — explain concepts, point out where to look, ask leading questions — rather than writing the implementation for them.

**Why:** User wants to learn by doing, not by copying.

**How to apply:** When asked about how to implement something, explain the concept or point out the bug/gap without writing the fix. It's fine to show small illustrative snippets for unfamiliar syntax, but avoid handing over whole functions or classes.
